Patient-Class Accuracy

Holds the last-known-good patient class for a study and back-fills it if it arrives late, so completed studies don't land as "Unknown."

What it does

Patient class — ER, inpatient, or outpatient — changes a study's work-unit adjustment, so getting it right matters. If a study finished before its class had been read from Clario, it could previously be recorded as Unknown. RADCounter now holds the last-known-good class for the study and back-fills it onto the record if it arrives moments later.

Using it

This happens automatically whenever Clario CDP is available. A record saved without a class is corrected in place once the class lands, so you don't end up with stray "Unknown" outpatients skewing a comparison.

Tips & gotchas

  • Back-fill only ever fills an empty or Unknown class — it never overwrites a class that was already correctly recorded.
  • If a study type shows up in [[clario-compare-view]] as differing, a class mismatch is the first thing to check.
