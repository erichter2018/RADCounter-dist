Clario CDP Reader

The connection RADCounter uses to read Clario directly — the per-study work unit, patient class, and priority.

What it does

RADCounter connects to Clario over the Chrome DevTools Protocol to read data the worklist already has: each study's work unit, its patient class (ER / inpatient / outpatient), and priority. This is what powers the Clario work-unit comparison and the patient-class accuracy fixes.

How to turn it on / set it up

Enable Settings → CDP → Enable Clario CDP (work-unit comparison) and run Set up Clario CDP, then launch Clario from the shortcut it creates. See [[cdp-setup]].

Using it

Once connected, the reader keeps the Clario link warm and polls periodically, so work units and signed times land shortly after you sign. The bottom-bar Clario CDP light shows the live connection state.

Tips & gotchas

  • The reader is read-only — it observes Clario, it doesn't drive it.
  • Patient class read here is what lets RADCounter back-fill a study that finished before its class was known — see [[patient-class]].
