Private Accession Handling

Accessions are stored hashed, never raw, and patient information is kept in memory only — never written to disk.

What it does

RADCounter is built to hold no protected health information at rest. Study accessions are stored as a hash, not the raw value, so records can still be matched and de-duplicated without keeping an identifier. Any patient information the app touches lives in memory only and is never persisted.

Using it

This is automatic and unconditional — there's nothing to switch on. It applies to the local database and to anything that gets backed up.

Tips & gotchas

  • Because hashing is deterministic, a repeated accession still matches (so duplicate-suppression works) without the raw value ever being stored.
  • The docs site itself contains no patient data of any kind.
